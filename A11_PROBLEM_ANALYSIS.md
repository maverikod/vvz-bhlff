# Детальный анализ проблемы теста A1.1 (Инвариантность к масштабу длины)

## Резюме проблемы

**Ошибка:** 141.42% (1.414214) - это указывает на фундаментальную проблему в логике сравнения решений.

## Ключевые находки из диагностики

### 1. **Корреляция решений = 0** ⚠️ КРИТИЧНО

```
Correlation <a1n, a2n_ds> = 0.000000e+00
```

Это означает, что нормализованные решения **ортогональны**, что физически невозможно для одинаковых безразмерных решений. Это указывает на **фундаментальную ошибку в методе сравнения**.

### 2. **Потеря спектральной структуры при downsampling**

```
|â1n[(4, 0, 0)]| = 1.000000e+00
|â2n[(4, 0, 0)]| = 0.000000e+00  ← ПОЛНАЯ ПОТЕРЯ!
```

После простого downsampling `a2n[::2, ::2, ::2]` спектральное представление **полностью теряется**. Это происходит потому, что:

- **Неправильный downsampling**: Простое взятие каждого второго элемента (`::stride`) **не соответствует** правильному downsampling в спектральной области
- **Правильный downsampling** должен:
  1. Взять FFT решения a2
  2. Извлечь низкочастотные компоненты (первые N1/2 мод по каждой оси)
  3. Применить обратное FFT к урезанному спектру

### 3. **Физическая проблема масштабирования**

Решения вычисляются на **разных физических областях**:
- Case 1: L=1.0, физические координаты x ∈ [0, 1)
- Case 2: L=2.0, физические координаты x ∈ [0, 2)

Даже при одинаковом безразмерном k* = 2π/L * m:
- Плоская волна с mode=4 на L=1 имеет **физическую длину волны** λ₁ = 1/4 = 0.25
- Плоская волна с mode=8 на L=2 имеет **физическую длину волны** λ₂ = 2/8 = 0.25

Хотя физические длины волн одинаковы, **структура решения на разных физических областях разная**:
- На L=1 укладывается 4 полных периода
- На L=2 укладывается 8 полных периодов

### 4. **Проблема сравнения на разных сетках**

Текущий подход:
1. Решить на сетке 128³ (L=1)
2. Решить на сетке 256³ (L=2)
3. Downsample второе решение простым `[::2, ::2, ::2]`
4. Сравнить нормализованные решения

**Проблема**: Простой downsampling не сохраняет спектральную структуру и приводит к потере информации.

## Корневые причины

### Причина 1: Неправильный алгоритм downsampling

**Текущий подход (неправильный):**
```python
a2n_ds = a2n[::stride, ::stride, ::stride]  # Простое прореживание
```

**Правильный подход должен быть:**
```python
# 1. FFT решения a2
a2_hat = ops.forward_fft(a2, "ortho")

# 2. Извлечь низкочастотные компоненты
# Для downsampling в 2 раза: взять первые N1 мод
a2_hat_ds = a2_hat[:N1, :N1, :N1]

# 3. Обратное FFT
a2_ds = ops.inverse_fft(a2_hat_ds, "ortho")
```

### Причина 2: Неправильная интерпретация требований

Согласно документу §A1.1:
> "Два прогона: (i) L=1,N=256; (ii) L=2,N=512 (тот же Δ). Один и тот же безразмерный источник s̃"

**Проблема**: "Один и тот же безразмерный источник" означает, что **физический источник должен быть масштабирован**, но текущая реализация использует:
- Case 1: mode=[4,0,0] → физический k = 2π/1 * 4 = 8π
- Case 2: mode=[8,0,0] → физический k = 2π/2 * 8 = 8π

Хотя физические k одинаковы, **спектральные представления разные** из-за разных размеров сетки.

### Причина 3: Проблема с нормировкой FFT

При разных L нормировка FFT может давать разные результаты:
- FFT с нормировкой "ortho" использует 1/√N для нормализации
- При разных N (128 vs 256) это может приводить к разным масштабам

## Рекомендации по исправлению

### Вариант 1: Правильный спектральный downsampling

```python
# Правильный downsampling через спектральную область
a2_hat = ops2.forward_fft(a2, "ortho")
# Извлечь низкочастотные компоненты
a2_hat_ds = a2_hat[:N1, :N1, :N1]
# Обратное FFT на меньшей сетке
ops1 = UnifiedSpectralOperations(domain1, precision="float64")
a2_ds = ops1.inverse_fft(a2_hat_ds, "ortho")
```

### Вариант 2: Сравнение в спектральной области

Вместо сравнения в физическом пространстве, сравнивать **спектральные коэффициенты**:

```python
# Нормализовать спектральные коэффициенты
a1_hat_norm = a1_hat / np.linalg.norm(a1_hat)
a2_hat_norm = a2_hat / np.linalg.norm(a2_hat)

# Сравнить соответствующие моды
# Для mode1 на L1 и mode2 на L2 должны быть одинаковые безразмерные k*
```

### Вариант 3: Использование интерполяции вместо downsampling

Вместо downsampling использовать **интерполяцию** решения a2 на сетку case 1:

```python
from scipy.interpolate import RegularGridInterpolator

# Создать интерполятор для a2
x2 = np.linspace(0, L2, N2, endpoint=False)
interp = RegularGridInterpolator((x2, x2, x2), a2)

# Интерполировать на сетку case 1
x1 = np.linspace(0, L1, N1, endpoint=False)
X1, Y1, Z1 = np.meshgrid(x1, x1, x1, indexing='ij')
points = np.stack([X1.ravel(), Y1.ravel(), Z1.ravel()], axis=1)
a2_interp = interp(points).reshape(N1, N1, N1)
```

### Вариант 4: Пересмотр требований теста

Возможно, тест должен проверять не точное совпадение решений, а **инвариантность безразмерных параметров**:
- Одинаковые безразмерные k*
- Одинаковые безразмерные амплитуды
- Одинаковые безразмерные фазы

## Выводы

1. **Основная проблема**: Неправильный алгоритм downsampling приводит к полной потере спектральной структуры
2. **Корреляция = 0**: Указывает на фундаментальную ошибку в методе сравнения
3. **Спектральная потеря**: После downsampling спектральное представление полностью теряется
4. **Рекомендация**: Использовать правильный спектральный downsampling или сравнение в спектральной области

## Следующие шаги

1. Реализовать правильный спектральный downsampling
2. Проверить корреляцию после исправления
3. Если проблема сохраняется, рассмотреть сравнение в спектральной области
4. При необходимости пересмотреть требования теста согласно физическому смыслу
