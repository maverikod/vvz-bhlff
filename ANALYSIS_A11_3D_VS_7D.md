## 1. Проблема #1: Использование 3D вместо 7D

### Текущая реализация

```python
def _create_3d_domain(L: float, N: int) -> Domain:
    return Domain(
        L=L,
        N=N,
        N_phi=1,  # ❌ Минимальные фазовые измерения
        N_t=1,    # ❌ Минимальное временное измерение
        T=1.0,
        dimensions=7,
    )
```

### Физическая проблема

**7D теория требует**:
- M₇ = ℝ³ₓ × 𝕋³_φ × ℝₜ
- **7D wave vector**: |k|² = |k_x|² + |k_φ|² + k_t²
- Все измерения важны для правильного решения

**При N_phi=1, N_t=1**:
- Фазовые измерения **не разрешены** (только k_φ=0)
- Временное измерение **не разрешено** (только k_t=0)
- Фактически решается **3D задача**, а не 7D!

### Математическая проблема

**7D спектральное решение**:
$$\widehat{a}(k_x, k_\phi, k_t) = \frac{\widehat{s}(k_x, k_\phi, k_t)}{\mu|k|^{2\beta} + \lambda}$$

где:
- **7D wave vector**: |k|² = |k_x|² + |k_φ|² + k_t²

**При N_phi=1, N_t=1**:
- k_φ = 0 (только одна мода, k_φ = 2π/1 * 0 = 0)
- k_t = 0 (только одна мода, k_t = 2π/1 * 0 = 0)
- |k|² = |k_x|² + 0 + 0 = |k_x|²

**Результат**: Решается **3D задача**, а не 7D!

### Проблема масштабирования

**Для 7D инвариантности** нужно масштабировать **все измерения**:
- Пространственные: L → 2L (проверяется)
- Фазовые: L_φ = 2π → должна масштабироваться тоже
- Временные: T → должна масштабироваться тоже

**При N_phi=1, N_t=1**:
- Фазовые и временные измерения **не масштабируются**
- Инвариантность проверяется **только для пространственных измерений**
- Это **неполная проверка** инвариантности!

## 2. Проблема #2: Не квадратная сетка

### Текущая реализация

```python
Domain(
    L=1.0, N=64,      # Пространственные: 64×64×64
    N_phi=1,          # Фазовые: 1×1×1 ❌
    N_t=1,            # Временные: 1 ❌
)
```

### Проблемы не квадратной сетки

#### 2.1. Проблема с FFT операциями

**FFT требует квадратной сетки** для правильной работы:
- При N_phi=1, FFT по фазовым измерениям тривиален (только DC компонента)
- При N_t=1, FFT по временному измерению тривиален (только DC компонента)
- Это может вызывать **численные артефакты** в спектральных операциях

#### 2.2. Проблема с downsampling

**При downsampling**:
- Case 1: (64, 64, 64, 1, 1, 1, 1)
- Case 2: (128, 128, 128, 1, 1, 1, 1)
- Downsampling: (128, 128, 128, 1, 1, 1, 1) → (64, 64, 64, 1, 1, 1, 1)

**Проблема**: Фазовые и временные измерения **не участвуют** в downsampling!
- Это может вызывать **искажения** при сравнении решений
- Downsampling работает только для пространственных измерений
- Фазовые и временные измерения остаются "замороженными" на DC компоненте

#### 2.3. Проблема с нормализацией FFT

**FFT нормализация "ortho"**:
- Для квадратной сетки: нормализация правильная
- Для не квадратной сетки: нормализация может быть неправильной
- При N_phi=1, N_t=1: нормализация может быть искажена

#### 2.4. Проблема с волновыми векторами

**7D wave vector computation**:
```python
k_phi = np.fft.fftfreq(N_phi, 2π/N_phi)  # При N_phi=1: только [0]
k_t = np.fft.fftfreq(N_t, T/N_t)        # При N_t=1: только [0]
```

**Проблема**:
- При N_phi=1: k_φ = [0] (только DC компонента)
- При N_t=1: k_t = [0] (только DC компонента)
- 7D wave vector: |k|² = |k_x|² + 0 + 0 = |k_x|²
- **Фактически решается 3D задача!**

## 3. Почему это критично для теста A1.1?

### Тест проверяет инвариантность к масштабу длины

**Требование**: При изменении L в 2 раза (L₁=1, L₂=2), решение должно быть безразмерно инвариантно.

**Проблема с 3D подходом**:
1. Проверяется инвариантность **только для пространственных измерений**
2. Фазовые и временные измерения **не проверяются**
3. Это **неполная проверка** инвариантности!

**Проблема с не квадратной сеткой**:
1. Downsampling работает **только для пространственных измерений**
2. Фазовые и временные измерения **не участвуют** в downsampling
3. Это может вызывать **искажения** при сравнении

## 4. Правильная реализация

### Должно быть:

```python
def _create_7d_domain(L: float, N: int) -> Domain:
    """
    Create proper 7D domain with square grid.
    
    Physical Meaning:
        Creates full 7D domain M₇ = ℝ³ₓ × 𝕋³_φ × ℝₜ
        with square grid for proper scale invariance testing.
    """
    return Domain(
        L=L,
        N=N,           # Пространственные: N×N×N
        N_phi=N,       # ✓ Фазовые: N×N×N (квадратная сетка!)
        N_t=N,         # ✓ Временные: N (квадратная сетка!)
        T=2*π,         # ✓ Период фазовых координат
        dimensions=7,
    )
```

### Почему квадратная сетка важна?

1. **Правильная FFT нормализация**: Все измерения имеют одинаковое разрешение
2. **Правильный downsampling**: Все измерения участвуют в downsampling
3. **Правильная проверка инвариантности**: Все измерения масштабируются одинаково
4. **Правильные волновые векторы**: Все измерения разрешены правильно

## 5. Математическое обоснование

### 7D wave vector для квадратной сетки

**При N_phi=N, N_t=N**:
- k_x: [0, ±1, ±2, ..., ±N/2] * 2π/L
- k_φ: [0, ±1, ±2, ..., ±N/2] * 2π/(2π) = [0, ±1, ±2, ..., ±N/2]
- k_t: [0, ±1, ±2, ..., ±N/2] * 2π/T

**7D wave vector**: |k|² = |k_x|² + |k_φ|² + k_t²

**Все измерения участвуют** в вычислении |k|!

### При N_phi=1, N_t=1:

- k_φ: [0] (только DC)
- k_t: [0] (только DC)
- |k|² = |k_x|² + 0 + 0 = |k_x|²

**Фактически решается 3D задача!**

## 6. Влияние на ошибку теста

### Текущая ошибка: 2.91e-02

**Возможные причины**:
1. ❌ Неправильное масштабирование источника (Root Cause #1)
2. ❌ Использование 3D вместо 7D (новая проблема!)
3. ❌ Не квадратная сетка (новая проблема!)
4. ❌ Ошибки в downsampling (Root Cause #2)

### Комбинированный эффект

**Использование 3D + не квадратная сетка** может давать:
- Искажения в спектральных операциях
- Неправильную нормализацию FFT
- Ошибки в downsampling
- Неполную проверку инвариантности

**Это может объяснить большую часть ошибки 2.91e-02!**
