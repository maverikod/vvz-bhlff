## 2. Root Cause Analysis

### Root Cause #1: Неправильное масштабирование амплитуды источника

#### Проблема

**Текущая реализация** (`test_A11_scale_length.py:89-94`):
```python
source_config = {
    "gaussian_amplitude": 1.0,  # ❌ НЕ масштабируется с L!
    "gaussian_width": 0.125,    # ✓ Правильно (нормализовано)
}
```

**Физический смысл**:
- Источник генерируется в нормализованных координатах [0, 1]
- `gaussian_width = 0.125` задается в нормализованных координатах
- Физическая ширина: σ_phys = L * σ_norm = L * 0.125 = L/8 ✓ (правильно)
- **НО**: Амплитуда источника **НЕ масштабируется** с L!

#### Математическое обоснование

Для безразмерной инвариантности решения уравнения:
$$\mathcal{L}_\beta a = s(x)$$

Если мы хотим, чтобы безразмерное решение $\tilde{a}(\tilde{x})$ было одинаковым для разных L, то:

1. **Безразмерные координаты**: $\tilde{x} = x/L$
2. **Безразмерный источник**: $\tilde{s}(\tilde{x}) = (L^3/A_0) \cdot s(L\tilde{x})$
3. **Масштабирование**: $s(x) = (A_0/L^3) \cdot \tilde{s}(x/L)$

Где $A_0$ - масштаб амплитуды.

**Для сохранения безразмерной нормы источника**:
- Если $\tilde{s}$ одинаков для обоих случаев
- То $s_1(x) = (A_0/L_1^3) \cdot \tilde{s}(x/L_1)$
- И $s_2(x) = (A_0/L_2^3) \cdot \tilde{s}(x/L_2)$
- Отношение: $s_2/s_1 = (L_1/L_2)^3 = (1/2)^3 = 1/8$

**Текущее поведение**:
- Амплитуда постоянна: `amplitude = 1.0` для обоих случаев
- Это означает, что физическая амплитуда не масштабируется
- Результат: безразмерные источники **различаются**!

#### Решение

```python
# В _solve_stationary:
amplitude_scale = 1.0 / (L**3)  # Масштабирование для безразмерной инвариантности
source_config = {
    "gaussian_amplitude": 1.0 * amplitude_scale,  # Масштабировать с L
    "gaussian_width": 0.125,  # Остается постоянным (нормализовано)
    ...
}
```

**Ожидаемый эффект**: Нормы источников должны стать одинаковыми (в безразмерных единицах).

### Root Cause #2: Ошибки в алгоритме spectral downsampling

#### Проблема

**Текущая реализация** (`test_A11_scale_length.py:212-227`):
```python
# Extract positive frequencies [0, 1, ..., N1//2-1]
a2_hat_ds[:N1//2, :N1//2, :N1//2] = a2_hat[:N1//2, :N1//2, :N1//2]

# Extract negative frequencies [-N1//2, ..., -2, -1]
# In FFT, these are stored at indices [N2-N1//2:N2]
a2_hat_ds[N1//2:, :N1//2, :N1//2] = a2_hat[N2-N1//2:, :N1//2, :N1//2]
# ... (8 октантов)
```

#### Анализ FFT frequency layout

**Для N2=128**:
- FFT frequencies: `[0, 1, 2, ..., 63, -64, -63, ..., -2, -1]`
- Indices: `[0:64]` (positive), `[64:128]` (negative)
- Negative frequencies stored as: `[64:128] = [-64, -63, ..., -1]`

**Для N1=64**:
- FFT frequencies: `[0, 1, 2, ..., 31, -32, -31, ..., -2, -1]`
- Indices: `[0:32]` (positive), `[32:64]` (negative)
- Negative frequencies stored as: `[32:64] = [-32, -31, ..., -1]`

**Правильное отображение**:
- Positive: `a2_hat[0:N1//2, ...]` → `a2_hat_ds[0:N1//2, ...]` ✓
- Negative: `a2_hat[N2-N1//2:N2, ...]` → `a2_hat_ds[N1//2:N1, ...]`

**Проблема**: Для 3D случая нужно правильно обрабатывать все 8 октантов, но текущая реализация может иметь ошибки в индексации для комбинаций осей.

#### Дополнительные проблемы

1. **Нормализация FFT**: Используется "ortho" нормализация, но при downsampling нормализация может измениться
2. **Фазовое выравнивание**: Фазовое выравнивание через корреляцию может быть недостаточным для точного сравнения
3. **Ошибки округления**: Накопление ошибок при обратном FFT после downsampling

#### Решение

**Вариант 1**: Улучшить алгоритм downsampling
- Использовать правильное отображение частот через `np.fft.fftfreq`
- Проверить нормализацию FFT
- Обработать все 8 октантов корректно

**Вариант 2**: Сравнивать напрямую в спектральном пространстве (РЕКОМЕНДУЕТСЯ)
- Вместо downsampling и сравнения в физическом пространстве
- Сравнивать соответствующие моды напрямую в спектральном пространстве
- Это избежит ошибок интерполяции и downsampling

### Root Cause #3: Сравнение в физическом пространстве вместо спектрального

#### Проблема

**Текущий подход**:
1. Downsample a2_hat → a2_hat_ds (спектральный)
2. Inverse FFT: a2_hat_ds → a2_ds (физический)
3. Сравнение: a1 vs a2_ds (физический)

**Проблемы**:
- Ошибки интерполяции при downsampling
- Ошибки округления при обратном FFT
- Накопление ошибок численных операций
- Фазовое выравнивание может быть неточным

#### Решение

**Сравнение в спектральном пространстве**:
1. Извлечь соответствующие моды из a1_hat и a2_hat
2. Сравнить напрямую в спектральном пространстве
3. Избежать downsampling и обратного FFT

**Преимущества**:
- Нет ошибок интерполяции
- Нет ошибок обратного FFT
- Прямое сравнение спектральных коэффициентов
- Выше точность

---

### Root Cause #4: Разные нормы из-за разных размеров сетки

#### Проблема

Даже при правильном масштабировании, нормы решений могут различаться из-за:
- Разных размеров сетки (N1=64 vs N2=128)
- Разных объемов интегрирования (L₁³ vs L₂³)
- Разных численных ошибок на разных сетках

#### Решение

**Нормализация для сравнения**:
- Использовать безразмерные нормы
- Нормализовать по объему: `norm_dimless = norm / (L³)`
- Или сравнивать нормализованные решения: `a_norm = a / ||a||`

## Приложение: Математические детали

### Безразмерная инвариантность

Для уравнения:
$$\mathcal{L}_\beta a = \mu(-\Delta)^\beta a + \lambda a = s(x)$$

**Безразмерные переменные**:
- $\tilde{x} = x/L$ (безразмерные координаты)
- $\tilde{a}(\tilde{x}) = a(L\tilde{x}) / A_0$ (безразмерное решение)
- $\tilde{s}(\tilde{x}) = s(L\tilde{x}) \cdot L^3 / A_0$ (безразмерный источник)

**Условие инвариантности**:
Если $\tilde{s}$ одинаков для разных L, то $\tilde{a}$ должен быть одинаковым.

**Масштабирование источника**:
$$s(x) = \frac{A_0}{L^3} \cdot \tilde{s}\left(\frac{x}{L}\right)$$

Для сохранения безразмерной нормы: $A_0 \propto L^3$ или $s \propto 1/L^3$.
