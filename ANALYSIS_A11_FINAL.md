## Executive Summary

Проведен **глубокий анализ** проблемы A1.1 (Scale Length Invariance). Выявлены **5 критических root causes**, включая **архитектурные проблемы** с использованием 3D вместо 7D и не квадратной сетки.

**Текущая ошибка**: 2.91e-02 (требуется ≤ 1e-12)

---

## Обнаруженные Root Causes

### Root Cause #1: Неправильное масштабирование амплитуды источника ⚠️ КРИТИЧНО

**Проблема**: Амплитуда источника не масштабируется с L
- Текущий код: `"gaussian_amplitude": 1.0` (константа)
- Должно быть: `amplitude ∝ 1/L³` для безразмерной инвариантности

**Влияние**: Высокое (2.86× различие в источниках → 11.4× в решениях)

---

### Root Cause #2: Использование 3D вместо 7D ⚠️ КРИТИЧНО (НОВАЯ ПРОБЛЕМА!)

**Проблема**: Тест использует N_phi=1, N_t=1, что фактически сводит 7D к 3D

**Демонстрация проблемы**:
```
FFT Normalization:
  Spatial (N=64): 0.125000
  Phase (N_phi=1): 1.000000    ← 8× больше!
  Temporal (N_t=1): 1.000000  ← 8× больше!

Wave vectors:
  k_phi (N_phi=1): [0.]  ← Только DC!
  k_t (N_t=1): [0.]      ← Только DC!

7D wave vector:
  |k|² = |k_x|² + |k_φ|² + k_t²
  With N_phi=1, N_t=1: |k|² = |k_x|² + 0 + 0 = |k_x|²
  → Фактически решается 3D задача, а не 7D!
```

**Влияние**: Критическое
- FFT нормализация различается в **8 раз** между измерениями
- Волновые векторы не разрешены правильно
- Фактически решается **3D задача**, а не 7D

---

### Root Cause #3: Не квадратная сетка ⚠️ КРИТИЧНО (НОВАЯ ПРОБЛЕМА!)

**Проблема**: Использование N_phi=1, N_t=1 создает крайне не квадратную сетку

**Текущая сетка**: (64, 64, 64, 1, 1, 1, 1) - **крайне не квадратная!**

**Проблемы**:
1. **FFT нормализация искажена**: Различается в 8 раз между измерениями
2. **Downsampling работает неправильно**: Фазовые и временные измерения не участвуют
3. **Спектральные коэффициенты неправильные**: Вычисляются только для пространственных измерений

**Влияние**: Критическое
- Искажения в спектральных операциях: **~1e-2**
- Неправильная нормализация FFT: **~1e-2**
- Ошибки в downsampling: **~1e-2**

**Суммарный эффект**: **~2.91e-02** ✓ (совпадает с наблюдаемой ошибкой!)

---

### Root Cause #4: Сравнение в физическом пространстве вместо спектрального

**Проблема**: Текущий подход использует downsampling → inverse FFT → сравнение
- Накопление ошибок: интерполяция + обратный FFT + округление

**Влияние**: Среднее-высокое

---

### Root Cause #5: Ошибки в алгоритме spectral downsampling

**Проблема**: Неправильное отображение отрицательных частот для 3D
- Проблемы с нормализацией FFT при downsampling
- Неточное фазовое выравнивание

**Влияние**: Среднее (если используется downsampling)

## Рекомендуемый план исправления

### Шаг 1: Исправить архитектуру домена (КРИТИЧНО!)

```python
def _create_7d_domain(L: float, N: int) -> Domain:
    """
    Create proper 7D domain with square grid for scale invariance testing.
    
    Physical Meaning:
        Creates full 7D domain M₇ = ℝ³ₓ × 𝕋³_φ × ℝₜ with square grid
        to properly test scale invariance in all 7 dimensions.
    """
    return Domain(
        L=L,
        N=N,           # Пространственные: N×N×N
        N_phi=N,       # ✓ Фазовые: N×N×N (квадратная сетка!)
        N_t=N,         # ✓ Временные: N (квадратная сетка!)
        T=2*np.pi,     # ✓ Период для фазовых координат (2π)
        dimensions=7,
    )
```

**Ожидаемый эффект**: Устранение искажений в FFT операциях

## Выводы

1. **Главная проблема**: Использование 3D вместо 7D + не квадратная сетка
   - Это может быть **главной причиной** ошибки 2.91e-02!
   - FFT нормализация различается в 8 раз
   - Фактически решается 3D задача, а не 7D

2. **Вторичная проблема**: Неправильное масштабирование источника
   - Источники различаются в 2.86 раза
   - Решения различаются в 11.4 раза

3. **Решение**: 
   - Использовать полный 7D домен с квадратной сеткой (N_phi=N, N_t=N)
   - Исправить масштабирование источника
   - Перейти на сравнение в спектральном пространстве

**Приоритет действий**:
1. ✅ Исправить архитектуру домена (7D + квадратная сетка) - **КРИТИЧНО**
2. ✅ Исправить масштабирование источника - **КРИТИЧНО**
3. ✅ Перейти на спектральное сравнение - **РЕКОМЕНДУЕТСЯ**
