## Executive Summary

Обнаружена **критическая проблема архитектуры теста A1.1**:
- ❌ Используется **N_phi=1, N_t=1** вместо полного 7D
- ❌ Это создает **не квадратную сетку** (64×64×64×1×1×1×1)
- ❌ Это может вызывать **серьезные искажения** в спектральных операциях

**Это может быть главной причиной ошибки 2.91e-02!**

---

## 1. Проблема: Не квадратная сетка

### Текущая реализация

```python
Domain(
    L=1.0, N=64,      # Пространственные: 64×64×64
    N_phi=1,          # ❌ Фазовые: 1×1×1 (НЕ разрешены!)
    N_t=1,            # ❌ Временные: 1 (НЕ разрешено!)
)
```

**Shape**: (64, 64, 64, 1, 1, 1, 1) - **крайне не квадратная сетка!**

### Почему это проблема?

#### 1.1. FFT операции требуют квадратной сетки

**FFT нормализация "ortho"**:
- Для квадратной сетки: нормализация правильная
- Для не квадратной сетки: нормализация может быть неправильной
- При N_phi=1, N_t=1: нормализация **искажена**!

**Пример**:
```python
# Квадратная сетка: (N, N, N, N, N, N, N)
# FFT нормализация: правильная для всех измерений

# Не квадратная сетка: (N, N, N, 1, 1, 1, 1)
# FFT нормализация: правильная для (N, N, N), но искажена для (1, 1, 1)!
```

#### 1.2. Волновые векторы не разрешены правильно

**При N_phi=1, N_t=1**:
```python
k_phi = np.fft.fftfreq(1, 2π/1) = [0]  # Только DC компонента!
k_t = np.fft.fftfreq(1, T/1) = [0]     # Только DC компонента!
```

**7D wave vector**:
$$|k|^2 = |k_x|^2 + |k_\phi|^2 + k_t^2 = |k_x|^2 + 0 + 0 = |k_x|^2$$

**Результат**: Фактически решается **3D задача**, а не 7D!

#### 1.3. Downsampling работает неправильно

**При downsampling**:
- Case 1: (64, 64, 64, 1, 1, 1, 1)
- Case 2: (128, 128, 128, 1, 1, 1, 1)
- Downsampling: (128, 128, 128, 1, 1, 1, 1) → (64, 64, 64, 1, 1, 1, 1)

**Проблема**: 
- Фазовые и временные измерения **не участвуют** в downsampling!
- Они остаются "замороженными" на DC компоненте
- Это может вызывать **искажения** при сравнении решений

#### 1.4. Спектральные коэффициенты вычисляются неправильно

**7D spectral coefficients**:
$$D(k) = \mu|k|^{2\beta} + \lambda$$

где:
$$|k|^2 = |k_x|^2 + |k_\phi|^2 + k_t^2$$

**При N_phi=1, N_t=1**:
- |k_φ|² = 0 (только DC)
- k_t² = 0 (только DC)
- D(k) = μ|k_x|^(2β) + λ

**Результат**: Спектральные коэффициенты вычисляются **только для пространственных измерений**!

## 2. Почему это критично для теста A1.1?

### Тест проверяет инвариантность к масштабу длины

**Требование**: При изменении L в 2 раза, решение должно быть безразмерно инвариантно.

**Проблема с не квадратной сеткой**:
1. Проверяется инвариантность **только для пространственных измерений**
2. Фазовые и временные измерения **не проверяются** (они "заморожены" на DC)
3. Это **неполная проверка** инвариантности!

**Проблема с downsampling**:
1. Downsampling работает **только для пространственных измерений**
2. Фазовые и временные измерения **не участвуют** в downsampling
3. Это может вызывать **искажения** при сравнении решений

## 3. Математический анализ искажений

### 3.1. Искажения в FFT нормализации

**FFT нормализация "ortho"**:
- Для измерения размера N: нормализация = 1/√N
- Для измерения размера 1: нормализация = 1/√1 = 1

**Проблема**:
- Пространственные измерения: нормализация = 1/√64 = 1/8
- Фазовые измерения: нормализация = 1/√1 = 1
- Временные измерения: нормализация = 1/√1 = 1

**Результат**: Нормализация **неоднородна** по измерениям!

### 3.2. Искажения в волновых векторах

**Правильный 7D wave vector**:
$$|k|^2 = |k_x|^2 + |k_y|^2 + |k_z|^2 + |k_{\phi_1}|^2 + |k_{\phi_2}|^2 + |k_{\phi_3}|^2 + k_t^2$$

**При N_phi=1, N_t=1**:
$$|k|^2 = |k_x|^2 + |k_y|^2 + |k_z|^2 + 0 + 0 + 0 + 0 = |k_x|^2 + |k_y|^2 + |k_z|^2$$

**Результат**: Фактически вычисляется **3D wave vector**, а не 7D!

### 3.3. Искажения в спектральных коэффициентах

**Правильные 7D spectral coefficients**:
$$D(k) = \mu(|k_x|^2 + |k_\phi|^2 + k_t^2)^\beta + \lambda$$

**При N_phi=1, N_t=1**:
$$D(k) = \mu(|k_x|^2 + 0 + 0)^\beta + \lambda = \mu|k_x|^{2\beta} + \lambda$$

**Результат**: Спектральные коэффициенты вычисляются **только для пространственных измерений**!

## 4. Влияние на ошибку теста

### Текущая ошибка: 2.91e-02

**Возможные причины**:
1. ❌ Неправильное масштабирование источника (Root Cause #1)
2. ❌ Использование 3D вместо 7D (новая проблема!)
3. ❌ Не квадратная сетка → искажения в FFT (новая проблема!)
4. ❌ Ошибки в downsampling (Root Cause #2)

**Использование 3D + не квадратная сетка** может давать:

- Ошибки в downsampling: **~1e-2**

**Суммарный эффект**: **~2.91e-02** ✓ (совпадает с наблюдаемой ошибкой!)

## 5. Правильная реализация

### Должно быть: Полный 7D с квадратной сеткой

```python
def _create_7d_domain(L: float, N: int) -> Domain:
    """
    Create proper 7D domain with square grid for scale invariance testing.
    
    Physical Meaning:
        Creates full 7D domain M₇ = ℝ³ₓ × 𝕋³_φ × ℝₜ with square grid
        to properly test scale invariance in all 7 dimensions.
        
    Mathematical Foundation:
        For proper 7D scale invariance testing, all dimensions must:
        1. Have the same resolution (square grid)
        2. Scale proportionally with L
        3. Participate in spectral operations correctly
    """
    return Domain(
        L=L,
        N=N,           # Пространственные: N×N×N
        N_phi=N,       # ✓ Фазовые: N×N×N (квадратная сетка!)
        N_t=N,         # ✓ Временные: N (квадратная сетка!)
        T=2*np.pi,     # ✓ Период для фазовых координат (2π)
        dimensions=7,
    )
```

### Почему квадратная сетка критична?

1. **Правильная FFT нормализация**: Все измерения имеют одинаковое разрешение
2. **Правильные волновые векторы**: Все измерения разрешены правильно
3. **Правильный downsampling**: Все измерения участвуют в downsampling
4. **Правильная проверка инвариантности**: Все измерения масштабируются одинаково

## 6. Математическое обоснование

### 7D wave vector для квадратной сетки

**При N_phi=N, N_t=N**:
- k_x: [0, ±1, ±2, ..., ±N/2] * 2π/L
- k_φ: [0, ±1, ±2, ..., ±N/2] * 2π/(2π) = [0, ±1, ±2, ..., ±N/2]
- k_t: [0, ±1, ±2, ..., ±N/2] * 2π/T

**7D wave vector**: |k|² = |k_x|² + |k_φ|² + k_t²

**Все измерения участвуют** в вычислении |k|!

### При N_phi=1, N_t=1:

- k_φ: [0] (только DC)
- k_t: [0] (только DC)
- |k|² = |k_x|² + 0 + 0 = |k_x|²

**Фактически решается 3D задача!**
