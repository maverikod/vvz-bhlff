# Глубокий анализ проблемы A1.1: Root Causes

## Проблема
Тест A1.1 (Scale Length Invariance) падает с ошибкой 2.91e-02 вместо требуемых ≤ 1e-12.

## Наблюдаемые факты

### 1. Масштабирование источников
- Case 1 (L=1.0, N=64): ||s1||₂ = 1.24e+02
- Case 2 (L=2.0, N=128): ||s2||₂ = 3.54e+02
- **Проблема**: Источники различаются в 2.86 раза, хотя должны быть безразмерно инвариантны

### 2. Масштабирование решений
- Case 1: ||a1||₂ = 1.81e+00
- Case 2: ||a2||₂ = 2.07e+01
- **Проблема**: Решения различаются в 11.4 раза

### 3. Grid spacing
- Δ₁ = L₁/N₁ = 1.0/64 = 0.015625
- Δ₂ = L₂/N₂ = 2.0/128 = 0.015625
- **✓ Правильно**: Grid spacing совпадает

## Root Causes

### Причина 1: Неправильное масштабирование источника

**Проблема**: Гауссовский источник генерируется в нормализованных координатах [0, 1], но:
- `gaussian_width = 0.125` задается в нормализованных координатах
- Физическая ширина σ_phys = L * σ_norm = L * 0.125 = L/8 ✓ (правильно)
- **НО**: Амплитуда источника не масштабируется!

**Математика**:
- Для безразмерной инвариантности: если s̃(x̃) - безразмерный источник, то
- s(x) = (A₀/L³) * s̃(x/L), где A₀ - масштаб амплитуды
- При изменении L в 2 раза, амплитуда должна масштабироваться как 1/L³ для сохранения безразмерной нормы

**Текущая реализация**:
```python
source_config = {
    "gaussian_amplitude": 1.0,  # ❌ НЕ масштабируется с L!
    "gaussian_width": 0.125,    # ✓ Правильно (нормализовано)
}
```

**Решение**: Амплитуда должна масштабироваться как `amplitude ∝ 1/L³` для сохранения безразмерной нормы источника.

### Причина 2: Неправильный алгоритм spectral downsampling

**Проблема**: Текущий алгоритм извлечения низкочастотных компонентов имеет ошибки:

1. **Неправильное отображение отрицательных частот**:
   - Для N2=128, отрицательные частоты хранятся как: [64:128] = [-64, -63, ..., -1]
   - Для N1=64, нужны частоты: [-32, -31, ..., -1]
   - Текущий код: `a2_hat[N2-N1//2:, ...]` = `a2_hat[96:128, ...]`
   - Это правильно для первой оси, но для многомерного случая нужно правильно обрабатывать все комбинации

2. **Проблема с нормализацией FFT**:
   - Используется "ortho" нормализация
   - При downsampling нормализация может измениться
   - Нужно проверить, что нормализация сохраняется правильно

3. **Проблема с фазовым выравниванием**:
   - Фазовое выравнивание через корреляцию может быть недостаточным
   - Могут быть глобальные фазовые сдвиги из-за разных начальных условий

### Причина 3: Сравнение в физическом пространстве вместо спектрального

**Проблема**: Тест сравнивает решения в физическом пространстве после downsampling, что вносит дополнительные ошибки:
- Ошибки интерполяции при downsampling
- Ошибки округления при обратном FFT
- Накопление ошибок численных операций

**Лучший подход**: Сравнивать решения напрямую в спектральном пространстве для соответствующих частот, избегая downsampling.

### Причина 4: Разные нормы решений из-за разных размеров сетки

**Проблема**: Даже при правильном масштабировании, нормы решений могут различаться из-за:
- Разных размеров сетки (N1=64 vs N2=128)
- Разных объемов интегрирования
- Разных численных ошибок на разных сетках

**Решение**: Нужно сравнивать нормализованные решения или сравнивать в спектральном пространстве.

## Рекомендации по исправлению

### 1. Исправить масштабирование источника
```python
# В _solve_stationary:
amplitude_scale = 1.0 / (L**3)  # Масштабирование для безразмерной инвариантности
source_config = {
    "gaussian_amplitude": 1.0 * amplitude_scale,  # Масштабировать с L
    "gaussian_width": 0.125,  # Остается постоянным (нормализовано)
    ...
}
```

### 2. Улучшить алгоритм spectral downsampling
- Использовать правильное отображение частот через fftfreq
- Проверить нормализацию FFT
- Рассмотреть сравнение напрямую в спектральном пространстве

### 3. Сравнивать в спектральном пространстве
- Вместо downsampling и сравнения в физическом пространстве
- Сравнивать соответствующие моды напрямую в спектральном пространстве
- Это избежит ошибок интерполяции

### 4. Улучшить нормализацию
- Убедиться, что нормализация учитывает разные размеры сетки
- Использовать безразмерные нормы для сравнения

## Ожидаемый результат после исправлений

После исправления масштабирования источника и улучшения алгоритма сравнения, ошибка должна уменьшиться с 2.91e-02 до ≤ 1e-12.
