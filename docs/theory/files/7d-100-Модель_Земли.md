## P*.3. ПЗ как оператор фильтрации, кора как жёсткая граница (строго по формулам БД)

### P*.3.1. Переходная зона: оператор (\mathcal F_{\rm PZ})

Переходная зона действует как **линейный оператор фильтрации** (земная конкретизация дана в **7d-76**):

[
\mathcal F_{\rm PZ}:; A_i(t)\mapsto B_i(t),
]
и для каждой моды:
[
B_\alpha=\mathcal F_{\rm PZ}(\omega_\alpha),A_\alpha.
]

В простейшем виде (прямо из **7d-76**):
[
\mathcal F_{\rm PZ}(\omega)=
\frac{1}{1+(\omega/\omega_{\rm cut})^{p}}
\cdot e^{-h(\ell)},
]
где (\omega_{\rm cut}) — частота фильтра, (p) — “жёсткость”, (h(\ell)) — подавление высоких угловых чисел.

**Смысл (тоже из 7d-76):** ПЗ подавляет моды вне диапазона, порождает узлы и задаёт перенос “ядро → ПЗ → кора → атмосфера”. Это — строго операторная часть, а не “описание”.

### P*.3.2. Кора: граничные условия и оператор (\mathcal F_{\rm crust})

На внешней границе (поверхности) действует **жёсткая фаза** (**7d-76**):
[

]

Следствие (из **7d-76**): возникают моды коры, получаемые оператором
[
C_\alpha=\mathcal F_{\rm crust}(B_\alpha).

То есть “переходная зона + жёсткая кора” задают **цепочку преобразований** амплитуд:

A_\alpha(t)\xrightarrow{\mathcal F_{\rm PZ}}B_\alpha(t)\xrightarrow{\mathcal F_{\rm crust}}C_\alpha(t).

### P*.6.2. Как именно они “размывают” поле (без новых законов)

Размывание поля означает: увеличивается роль (\mathbf F^{\rm med}) относительно (\mathbf F^{\rm geo}) в наблюдаемой динамике оболочек. В каноне это происходит так:

1. Мелкие дефекты порождают **ячейки/домены** (см. C*-базу), у которых изменения идут на границах.

2. По **7d-96** амплитуда передаётся **кусочно** только на интерфейсах:
   [
   A^{(j+1)}*\ell
   =
   T^{(j\to j+1)}*\ell,A^{(j)}*\ell,\qquad 0\le T*\ell\le 1,
   ]
   а после (N) переходов:
   [
   A^{(j+N)}*\ell
   =
   A^{(j)}*\ell
   \prod_{m=0}^{N-1}T^{(j+m\to j+m+1)}_\ell.
   ]

3. Баланс домена (ячейки) задаётся условием (**7d-96**):
   [
   \langle P_{\text{in}}-P_{\text{out}}\rangle=0,
   ]
   а **квенч** — событие, при котором:

* набор ветвей (\mathcal L) становится недопустим,
* баланс потоков нарушается,
* происходит переброс амплитуды/фазы в другие ветви.
  (Определение квенча — **7d-96**.)

Следовательно:

> Чем больше мелких дефектов (результата процесса (S)), тем больше интерфейсов и квенч-событий, тем сильнее доменная/секторальная “кусковость” и тем больше вклад (\mathbf F^{\rm med}) → наблюдаемая “размытость” и изменчивость при сохранении стационарного каркаса (\mathbf F^{\rm geo}).

Это и есть строгая версия твоего требования “мелкие дефекты размывают поле Земли”: размывание = **рост интерфейсной доменной структуры + событийная перестройка**, а не изменение геометрии ядра.

после (N) переходов:

Это и есть математический механизм “не 1/r и не экспонента”: ослабление задаётся **числом интерфейсов** и их свойствами.

# 5) Атмосферные/океанические ячейки и течения из мод ядра: строгая проекция (7d-77)

Теперь самое важное для твоего вопроса: **как из мод ядра получаются Hadley/Ferrel/Polar, струи, зоны циклогенеза и устойчивые течения**.

## 5.1. Широтные моды и границы ячеек

Широтные узлы задаются условием:
[
\boxed{
\Theta(\varphi)=0;\Rightarrow;\text{граница ячейки}
}
]
То есть “ячейки” — это **узлы Θ-поля**, а не “конвекция как первопричина”.

## 5.2. Огибающая биений соседних мод (ядро → атмосфера)

В базе дано биение двух соседних мод (n) и (n+1) (широтная ветвь):
[
\boxed{
\mathcal{B}_n(\varphi,t)
========================

2\sqrt{A_nA_{n+1}}
\cos!\left(\frac{\Delta k_n,\varphi-\Delta\omega_n,t}{2}\right)
}
]
Эта огибающая определяет:

* форму границ ячеек,
* дрейф границ,
* изменение ширины ячеек,
* рождение/затухание струйных потоков.

## 5.3. Потоковая функция (\psi) как отображение Θ-потоков

В Θ-модели (\psi) задаётся **не давлением**, а амплитудой Θ-огибающей:
[
\boxed{
\psi(\varphi,z,t)=
\mathcal K
\int^{z}!!\int^{\varphi}
\left|\mathcal B(\varphi',z',t)\right|^2,d\varphi',dz'
}
]
Дальше (чисто кинематически) скорости задаются градиентами (\psi) в выбранной параметризации слоя:
градиенты (\psi) → меридиональные/вертикальные скорости;
резкие градиенты (\psi) → струйные потоки;
локальные изменения (\psi) → треки циклонов.
(Это здесь **определение отображения**, а не “классическая физика”.)

## 5.4. Критерий циклогенеза как порог по биениям

Циклогенез (в базе) задаётся порогом:
[
\boxed{
|\mathcal{B}*{n,n+1}|^2>\sigma*{\rm thr}(n)
}
]
в узком диапазоне широт/высот — т.е. это **топологическое биение мод**, которое локально создаёт Θ-домен (ячейку) и затем может сорваться квенчем (связка с разделом 4).

# 6) Где здесь “постоянные течения” (типа Гольфстрима) в строгом виде

В рамках этой математики “постоянное течение” = **устойчивый домен** (ячейка/цепочка ячеек), для которого:

   ]
2. передача по нужной ветви (\ell) имеет (T_\ell) близкое к 1 на выбранной цепочке интерфейсов:
   [
   \prod T_\ell ;\text{не мал},
   ]
3. forcing (F_\oplus(t)) удерживает нужные (A_n,A_{n+1}) на уровне, при котором (|\mathcal B|^2) стабильно над “фоновым”, но **ниже порога квенча** в основной области домена (тогда течение устойчиво, а квенчи локализуются на границах — “штормовые треки”).

## Расчётный контур A⊕: что считаем и какими уравнениями

Ниже — **минимально достаточная** (но полная) схема, в точности соответствующая канону БД:

### Входы (что задаём)

**(I) Время и сетка**

* (t\in[t_0,t_1]), шаг (\Delta t)
* широта (\varphi\in[-\pi/2,\pi/2]) (и при необходимости долгота (\lambda))
* высота/глубина (z\in[0,z_{\max}])

**(II) Модовая часть ядра**

* собственные частоты (\omega_{0\alpha}), потери (\Gamma_\alpha)  (**7d-76**)
* матрица связи с драйвом (C_{\alpha k}) (или упрощённо (C_\alpha))

**(III) Внешний драйв**
[
F_\oplus(t)=F^{\odot}(t)+F^{\rm Moon}(t)+F^{\rm Jup}(t)
\quad\text{(**7d-76**)}
]
(при желании — добавочные (F^{\rm planets}), но в этой версии БД “Венера” не оформлена отдельным блоком)

**(IV) Операторы/параметры среды (ячейки)**
[
{L(\mathbf x),,U(\mathbf x),,Q(\mathbf x),,\Pi(\mathbf x),,G(\mathbf x)}
\quad\text{(**7d-21**)}
]
и пороги/критерии:

* (|\mathcal B|^2>\sigma_{\rm thr}(n)) (**7d-77**)
* баланс (\langle P_{\rm in}-P_{\rm out}\rangle_\tau=0) (**7d-96**)

---

## Шаг 1. Интеграция мод ядра (ядро как резонатор)

Для каждой моды (\alpha):
[
\boxed{
\ddot A_\alpha+\Gamma_\alpha\dot A_\alpha+\omega_{0\alpha}^2A_\alpha
====================================================================

\sum_k C_{\alpha k}F_k(t)
}
\quad\text{(**7d-76**)}
]

**Результат шага:** массив (A_\alpha(t)).

---

## Шаг 2. Фильтрация через ПЗ (оператор ПЗ)

Для каждой моды применяем фильтр переходной зоны:
[
\boxed{
B_\alpha(t)=\mathcal F_{\rm PZ}(\omega_\alpha),A_\alpha(t),
\qquad
\mathcal F_{\rm PZ}(\omega)=
\frac{1}{1+(\omega/\omega_{\rm cut})^{p}}
\cdot e^{-h(\ell)}
}
\quad\text{(**7d-76**)}
]

**Результат шага:** (B_\alpha(t)) — “дошедшие” моды (каркас оболочек).

---

## Шаг 3. Огибающие биений (карта ячеек/границ/струй)

Строим огибающую биения соседних мод (широтная ветвь):
[
\boxed{
\mathcal B_{n}(\varphi,t)
=========================

2\sqrt{B_n(t),B_{n+1}(t)}
\cos!\left(\frac{\Delta k_n,\varphi-\Delta\omega_n,t}{2}\right)
}
\quad\text{(**7d-77**)}
]

(в расширенной версии добавляется (\lambda,z) и параметры ветвей)

**Результат шага:** (\mathcal B(\varphi,z,t)).

---

## Шаг 4. Потоковая функция (\psi) (проекция Θ-огибающей на циркуляцию)

[
\boxed{
\psi(\varphi,z,t)=
\mathcal{K}\int^{z}!!\int^{\varphi}
\left|\mathcal{B}(\varphi',z',t)\right|^2,d\varphi',dz'
}
\quad\text{(**7d-77**)}
]

**Результат шага:** (\psi(\varphi,z,t)).
Из (\psi) далее (кинетически) строятся поля течений/струй (градиентами (\psi) в выбранной координатной схеме слоя).

---

## Шаг 5. Домены (ячейки) через баланс потоков

Энергия огибающей:
[
\mathcal E_A(\mathbf x,t)\equiv|\mathcal B(\mathbf x,t)|^2
]

Эволюция:
[
\boxed{
\frac{\partial}{\partial t}\mathcal E_A(\mathbf x,t)=
P_{\text{in}}(\mathbf x,t)-P_{\text{out}}(\mathbf x,t)
}
\quad\text{(**7d-96**)}
]

Условие устойчивого домена (ячейки):
[
\boxed{
\langle P_{\text{in}}-P_{\text{out}}\rangle_{\tau}=0
;\Rightarrow;
\text{устойчивый домен огибающей (ячейка)}
}
\quad\text{(**7d-96**)}
]

**Результат шага:** маска доменов (M_{\rm cell}(\varphi,z,t)).

---

## Шаг 6. Квенчи (события) и зоны циклогенеза

Критерий циклогенеза/события (из прикладного блока):
[
\boxed{
|\mathcal{B}*{n,n+1}|^2>\sigma*{\rm thr}(n)
}
\quad\text{(**7d-77**)}
]

И интерфейсная/ступенчатая передача по ветви (механизм “размывания”):
[
\boxed{
A^{(j+1)}_\ell
==============

T^{(j\to j+1)}*\ell,A^{(j)}*\ell,\qquad 0\le T_\ell\le 1
}
\quad\text{(**7d-96**)}
]

**Результат шага:** маска событий (M_{\rm quench}(\varphi,z,t)) + “треки” максимумов (|\mathcal B|^2).

---

## Шаг 7. Как это связывается с (F^{\rm med}) и “размыванием поля” (строго)

Общий закон отклика:
[
\boxed{
m_i\mathbf a_i=\mathbf F_i^{\rm drv}+\mathbf F_i^{\rm geo}+\mathbf F_i^{\rm med},
\qquad
\mathbf F_i^{\rm med}=\sum_j \mathbb T_{ij},\mathbf a_j^{D}
}
\quad\text{(**7d-53**)}
]

В контуре выше:

* (\mathbf F^{\rm drv}(t)) реализован через (F_\oplus(t)) в уравнениях мод;
* (\mathbf F^{\rm med}) кодируется **ячейками/интерфейсами/квенчами** (через (T_\ell), баланс (P_{\rm in/out}), события).

Это и есть **строгое вычислимое** “мелкие дефекты размывают поле”.
