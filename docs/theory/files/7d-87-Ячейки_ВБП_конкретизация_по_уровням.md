## 3.1. Напоминание

Локальный функционал уровня уже задан как :

[
I_\ell(\mathbf r)
=================

\mathcal{I}_\ell\Big(
\Phi^{(\ell)}(\mathbf r),
\nabla \Phi^{(\ell)}(\mathbf r),
\Delta \Phi^{(\ell)}(\mathbf r), \dots
\Big),
]

где (\Phi^{(\ell)}) — вклад Θ-поля уровня (\ell) в полное поле (\Phi(\mathbf r)=\sum_\ell \Phi^{(\ell)}(\mathbf r)).

Дальше конкретизируем (\mathcal I_\ell) для выбранных уровней.

---

## 3.2. Конкретизация (I_\ell) по уровням

### 3.2.1. Галактика ((\ell=2))

Для галактики нас интересует **выделение ячейки стенок/плато** по кривым вращения SPARC и её DCT-спектру. Введём радиальную координату (R) в плоскости диска, и Θ-потенциал уровня галактики (\Phi^{(2)}(R,\varphi,z)).

Определим локальную «энергетическую плотность стены» как функцию от кривизны и градиента:

[
\varepsilon_{\rm wall}^{(2)}(R)
===============================

a_1 \left|\frac{\partial^2 \Phi^{(2)}}{\partial R^2}\right|
+
a_2 \left|\frac{\partial \Phi^{(2)}}{\partial R}\right|
]

(коэффициенты (a_{1,2}) — нормировки).

Пусть (\mathcal{S}^{(2)}(k;R)) — локальный DCT/Фурье-спектр Θ-поля/скоростей по радиусу для данного (R) (как в SPARC-анализе), а (\mathcal{W}_{\rm core}(k)) — окно частот, соответствующее основным семействам ядерных мод диска (C/S-семейства). Тогда:

[
E_{\rm core}^{(2)}(R)
=====================

\int \mathcal{W}_{\rm core}(k),
\mathcal{S}^{(2)}(k;R),dk.
]

Тогда функционал уровня:

[
\boxed{
I_2(R)
======

f_2\Big(
\varepsilon_{\rm wall}^{(2)}(R),
E_{\rm core}^{(2)}(R)
\Big)
=====

w_1,\varepsilon_{\rm wall}^{(2)}(R)
+
w_2,E_{\rm core}^{(2)}(R),
}
]

где (w_{1,2}) — весовые коэффициенты.

Смысл: ячейка на галактическом уровне существует там, где одновременно сильны **радиальные «стены»** (резкая кривизна профиля) и **энергия базовых семейства DCT-мод**, соответствующих торовой структуре галактики.

---

### 3.2.2. Солнце ((\ell=4))

Для Солнца в главе S введена структура как трёхтровый Θ-дефект (\mathcal D_\odot=(T_1,T_2,T_3;W)_\odot). Пусть

[
\Theta_\odot(\mathbf x,t)
=========================

\sum_\alpha a_\alpha(t),\Psi_\alpha(\mathbf x)
]

— разложение по солнечным модам (ядерные, радиативные, конвективные, корональные).

Определим локальную энергию мод:

[
E_\odot(\mathbf x,t)
====================

\sum_\alpha \rho_\alpha,|a_\alpha(t),\Psi_\alpha(\mathbf x)|^2,
]

где (\rho_\alpha) задаёт вклад класса мод (ядро, T₁; радиативная зона, T₂; конвекция/EM-трубки, T₃ и хвост).

Для выделения **солнечных ячеек** (пятна, трубки, активные области) нам нужна ещё мера **анизотропии**/«трубчатости»:

[
A_\odot(\mathbf x,t)
====================

## \left|\nabla_\parallel \Theta_\odot\right|^2

\left|\nabla_\perp \Theta_\odot\right|^2,
]

где (\nabla_\parallel) и (\nabla_\perp) — производные вдоль/поперёк локальных магнитных/трубчатых структур (задаётся геометрией мод из главы S).

Тогда:

[
\boxed{
I_4(\mathbf x,t)
================

f_4\Big(
E_\odot(\mathbf x,t),
A_\odot(\mathbf x,t)
\Big)
=====

w_E^{(\odot)}E_\odot(\mathbf x,t)
+
w_A^{(\odot)}|A_\odot(\mathbf x,t)|.
}
]

Смысл: ячейка уровня звезды (солнечная активная область) возникает там, где локальная энергия мод велика и профиль сильно анизотропен (трубка/пятно, протуберанец).

---

### 3.2.3. Земля ((\ell=6,7))

Для Земли уже выписано полное уравнение мод:

[
\Theta_\oplus(\mathbf x,t)
==========================

\sum_\alpha A_\alpha(t),\Phi_\alpha(\mathbf x),
]

[
\ddot A_\alpha + \Gamma_\alpha \dot A_\alpha + \omega_{0\alpha}^2 A_\alpha
==========================================================================

\sum_\beta K_{\alpha\beta} F_\beta(t),
]

где (F_\beta(t)) включает солнечный, лунный, юпитерианский драйверы, а фильтры (\mathcal F_{\rm PZ}, \mathcal F_{\rm crust}) задают роль ПЗ и коры.

1. **Выделение «эффективных» поверхностных мод.**
   Пусть (B_\alpha = \mathcal F_{\rm PZ}(\omega_\alpha) A_\alpha) и (C_\alpha = \mathcal F_{\rm crust}(B_\alpha)) — амплитуды после фильтра ПЗ и коры. Тогда локальное поле на уровне коры/атмосферы:

   [
   \Theta_\oplus^{\rm surf}(\mathbf x,t)
   =====================================

   \sum_\alpha C_\alpha(t),\Phi_\alpha^{\rm surf}(\mathbf x).
   ]

2. **Энергия ячейки Земли.**

   [
   E_\oplus(\mathbf x,t)
   =====================

   \sum_\alpha \rho_\alpha^{(\oplus)},|C_\alpha(t),\Phi_\alpha^{\rm surf}(\mathbf x)|^2,
   ]

   где веса (\rho_\alpha^{(\oplus)}) выбираются так, чтобы:

   * глубокие моды ядра/ПЗ слабо видимы на поверхности;
   * моды коры/атмосферы (циклоны, ячейки Хэдли/Феррела, сейсмика) доминировали.

3. **Градиентная структура (границы ячеек).**

   [
   G_\oplus(\mathbf x,t)
   =====================

   \big|\nabla_\perp \Theta_\oplus^{\rm surf}\big|^2
   +
   \lambda,\big|\Delta \Theta_\oplus^{\rm surf}\big|,
   ]

   где (\nabla_\perп) — градиент в горизонтальной плоскости; (\lambda) — вес вторых производных (границы фронтов, ячеек).

Тогда функционал:

[
\boxed{
I_\oplus(\mathbf x,t)
\equiv
I_{6,7}(\mathbf x,t)
====================

f_{6,7}\Big(
E_\oplus(\mathbf x,t),
G_\oplus(\mathbf x,t)
\Big)
=====

w_E^{(\oplus)}E_\oplus
+
w_G^{(\oplus)}G_\oplus.
}
]

Смысл: ячейка земного уровня (циклон, зона тектонического напряжения, резонансная область) — там, где одновременно велика **локальная энергия фильтрованных мод** и **сильны горизонтальные градиенты/кривизна**.

---

### 3.2.4. Протон ((\ell=9))

Протон — трёхтровый дефект уровня (N\sim \text{ядерный}) с обмоткой и B=1 (см. главу о протоне/частицах — она в индексе рядом с 7d-85). Пусть (\Theta_p(\mathbf x)) — стационарный профиль Θ-поля протона.

Определим:

1. **Энергетическая плотность:**

   [
   E_p(\mathbf x)
   ==============

   \frac12
   \left(
   |\nabla \Theta_p|^2
   +
   \mu,|\Delta \Theta_p|
   \right),
   ]

   где (\mu) задаёт вклад «жёсткости» конфигурации.

2. **Топологический инвариант (плотность барионного заряда):**

   в схеме FR-кручения вводится псевдоскалярная плотность (q_B(\mathbf x)), зависящая от векторизованного Θ-поля, условно:

   [
   q_B(\mathbf x)
   \sim
   \epsilon^{ijk}
   \partial_i \hat\Theta,
   (\partial_j \hat\Theta \times \partial_k \hat\Theta),
   ]

   где (\hat\Theta) — нормированное поле на многообразии мод.

Тогда:

[
\boxed{
I_9(\mathbf x)
==============

f_9\Big(
E_p(\mathbf x),
|q_B(\mathbf x)|
\Big)
=====

w_E^{(p)}E_p(\mathbf x)
+
w_B^{(p)}|q_B(\mathbf x)|.
}
]

Смысл: ячейка протонного уровня существует там, где **конфигурация энергетически локализована** и одновременно несёт **устойчивый топологический заряд**.

## 3.3. Связь (I_\oplus) с уравнением земных мод

Уравнение земных мод уже собрано в виде:

[
\ddot A_\alpha
+
\Gamma_\alpha\dot A_\alpha
+
\omega_{0\alpha}^2 A_\alpha
===========================

\sum_\beta K_{\alpha\beta} F_\beta(t),
]

[
\Theta_\oplus(\mathbf x,t)
==========================

\sum_\alpha A_\alpha(t),\Phi_\alpha(\mathbf x),
]

с фильтрами ПЗ и коры (\mathcal F_{\rm PZ},\mathcal F_{\rm crust}).

Функционал (I_\oplus) выше — это **чисто производная величина от решения этой системы**:

1. Сначала решается система для (A_\alpha(t)).
2. Применяются фильтры ПЗ и коры:
   [
   B_\alpha = \mathcal F_{\rm PZ}(\omega_\alpha)A_\alpha,\quad
   C_\alpha = \mathcal F_{\rm crust}(B_\alpha).
   ]
3. Строится (\Theta_\oplus^{\rm surf}(\mathbf x,t)) и по нему — (E_\oplus, G_\oplus).
4. Подставляется в определение (I_\oplus(\mathbf x,t)).

То есть для уровня Земли **никаких дополнительных уравнений** не вводится: (I_\oplus) — функционал от ({A_\alpha(t)}), найденных из уже выписанного «полного уравнения земных мод».

## 3.4. Связь (I_4) с моделью Солнца

В главе S Солнце задано как трёхтровый дефект (\mathcal D_\odot=(T_1,T_2,T_3;W)_\odot) с разбиением на ядро, радиативную зону, конвекцию и корону. Внутри главы (по аналогии с Землёй) вводится разложение:

[
\Theta_\odot(\mathbf x,t)
=========================

\sum_\alpha a_\alpha(t),\Psi_\alpha(\mathbf x),
]

где классы мод отвечают:

* ядру (генерация энергии/нейтрино),
* радиативной зоне (фильтрация),
* конвективной зоне и короне (EM-трубки, пятна, протуберанцы).

Функционал (I_4(\mathbf x,t)) выше — это:

[
I_4(\mathbf x,t)
================

w_E^{(\odot)}\sum_\alpha \rho_\alpha |a_\alpha(t)\Psi_\alpha(\mathbf x)|^2
+
w_A^{(\odot)}|A_\odot(\mathbf x,t)|,
]

где (A_\odot) построено из градиентов Θ-поля.

Т.е. как и у Земли, **динамика задаётся уравнениями мод Солнца** (аналог земного уравнения, но с другим спектром и драйверами), а (I_4) — индикатор «ячейки активности» (пятно, трубка), построенный на решении этих уравнений.

## 3.5. Общее мультиуровневое уравнение

Для цепочки **Вселенная → галактика → Солнце → Земля → протон** удобно ввести для каждого уровня (\ell):

* поле (\Phi^{(\ell)}(\mathbf x,t)),
* модовые амплитуды (A_{\ell n}(t)),
* собственные частоты (\omega_{\ell n}),
* коэффициенты затухания (\Gamma_{\ell n}),
* связь с вышележащими уровнями через драйвер (F_{\ell n}^{\rm ext}(t)).

Универсальная форма:

[
\boxed{
\ddot A_{\ell n}(t)
+
\Gamma_{\ell n}\dot A_{\ell n}(t)
+
\omega_{\ell n}^2 A_{\ell n}(t)
===============================

\sum_m K_{\ell n,m},
F_m^{(\ge \ell)}(t),
}
]

[
\Phi^{(\ell)}(\mathbf x,t)
==========================

\sum_n A_{\ell n}(t),\Psi_{\ell n}(\mathbf x).
]

Здесь:

* для **протона** ((\ell=9)) внешних драйверов почти нет: колебания заданы внутренним спектром;
* для **Земли** ((\ell=6,7)) драйверы идут от Солнца, Луны, Юпитера и, через фон Θ, от галактики;
* для **Солнца** ((\ell=4)) драйверы идут от галактического поля и планет;
* для **галактики** ((\ell=2)) драйверы идут от кластера и макрофона (через операторы (P_{\rm tail}, P_{\rm cluster},P_{\rm macro}) — они уже описаны в главе о кластерах/макроуровне).

На каждом уровне определён свой (\mathcal I_\ell) и, следовательно, (I_\ell(\mathbf r,t)). Совместный индикатор ячейки уже задан:

[
I_{\text{cell}}(\mathbf r,t)
============================

\prod_{\ell\in S}
g_\ell\big(I_\ell(\mathbf r,t)\big),
]

с условием существования ячейки (I_{\rm cell}\ge I_{\rm thr}).

## 3.6. ТЗ для единого вычислительного комплекса

На базе общего уравнения и функционалов (I_\ell) ТЗ для ПО можно сформулировать так (в сжатом виде, чтобы дальше развернуть в отдельной главе):

### 3.6.1. Цепочка уровней и общая схема

1. **Уровни, которые обязаны поддерживаться:**

   * (\ell=0,1): макрофон и кластер (фоновые поля, задаваемые внешними данными CMB/линзирование).
   * (\ell=2): конкретная галактика (SPARC).
   * (\ell=4): Солнце (гелиосейсмология, данные по циклам).
   * (\ell=6,7): Земля (сейсмика, ERA5, циклоны).
   * (\ell=9): протон (лабораторные параметры).

2. **Для каждого уровня комплекс обязан:**

   * хранить разложение (\Phi^{(\ell)} = \sum_n A_{\ell n}(t)\Psi_{\ell n}(\mathbf x));
   * интегрировать систему динамики (A_{\ell n}(t)) в универсальном виде;
   * предоставлять реализацию (\mathcal I_\ell) и вычислять (I_\ell(\mathbf r,t));
   * экспортировать/импортировать драйверы (F_{\ell n}^{\rm ext}(t)) для связи с соседними уровнями.

---

### 3.6.2. Модульная структура

**1. Universe/Cluster Module**

* Вход: спектры Θ-поля по CMB, крупномасштабное линзирование.
* Выход: (\Theta_{\rm macro},\Theta_{\rm cluster}), поля (\Theta_{\rm ext}^{(i)}) для галактик (как в X.8–X.10).
* Интерфейс: предоставляет драйверы для уровня (\ell=2).

**2. Galaxy Module (SPARC)**

* Вход: кривые вращения V(R), DCT-спектры.
* Задача:

  * подбирать (\omega_{2n},\Gamma_{2n}) и формы (\Psi_{2n}) под данные;
  * считать (I_2(R)) по формуле выше.
* Выход:

  * поля (\Phi^{(2)}), драйверы для (\ell=4) (солнечная система).

**3. Sun Module**

* Вход: (\Theta^{(2)}) как внешний фон, планетарные драйверы.
* Задача:

  * интегрировать уравнения мод Солнца (аналог земного, но с иным спектром);
  * вычислять (I_4(\mathbf x,t)) (карты активных ячеек).
* Выход:

  * драйверы (F_\oplus^{\rm Sun}(t)) для Земли;
  * карты солнечных ячеек для сравнения с пятнами/протуберанцами.

**4. Earth Module**

* Вход: драйверы от Солнца/Луны/Юпитера, фон от галактики.
* Задача:

  * интегрировать «полное уравнение земных мод»;
  * применять (\mathcal F_{\rm PZ},\mathcal F_{\rm crust});
  * вычислять (I_\oplus(\mathbf x,t)).
* Выход:

  * карты земных ячеек (циклоны, зоны напряжений);
  * прогнозы по циклоногенезу, сейсмике.

**5. Proton Module**

* Вход: параметры трёхтрового протона из теоретической главы.
* Задача:

  * решать стационарную задачу для (\Theta_p(\mathbf x));
  * считать (I_9(\mathbf x)) и интегральные величины (масса, заряд).
* Выход:

  * эталонные параметры для масштабирования к Солнцу и другим уровням.

**6. Indicator/Cell Module**

* Принимает (I_\ell(\mathbf r,t)) с разных уровней, собирает (I_{\rm cell}(\mathbf r,t)) по заданным множествам (S) (например, для атмосферной ячейки Земли (S={0,2,4,5,6,7}) уже упомянуто в 86-м блоке).
* Реализует порог (I_{\rm thr}) и строит маску (\chi_{\rm cell}).

---

### 3.6.3. Общие требования к реализации

1. **Единый интерфейс уровня (\ell)**
   Каждый модуль реализует один и тот же абстрактный интерфейс:

   * `solve_modes(dt, T)` — интеграция системы (\ddot A+\Gamma\dot A+\omega^2A=K F),
   * `get_field(x,t)` — возвращает (\Phi^{(\ell)}(\mathbf x,t)),
   * `get_indicator(x,t)` — возвращает (I_\ell(\mathbf x,t)),
   * `get_drivers(t)` — даёт драйверы для нижележащих уровней.

2. **Строгое следование Θ-подходу:**

   * никаких параметров типа плотности/температуры/плазмы (как уже запрещено для CMB);
   * материя и наблюдаемые величины — только как вторичные карты, построенные из (\Phi^{(\ell)}) и (I_\ell).

3. **Масштабируемость:**

   * реализация должна позволять легко добавить промежуточные уровни (ядро → звезда → галактика → кластер и т.п.), используя ту же форму уравнений и функционалов.
