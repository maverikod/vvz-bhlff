Ниже — **конкретное ТЗ для модели-оцифровщика**: как **прочитать статьи**, составить **карту извлекаемых экспериментальных данных**, затем **распознать/оцифровать** и **проверить**. Это можно копировать целиком как промпт для локального запуска.

# ТЗ: оцифровка экспериментальных данных из статей (кристаллы)

## 0) Общая цель

Из каждой статьи извлечь **только экспериментальные данные**, пригодные для прямого сравнения (табличные и графические наборы):

* **фононные дисперсии**: (\omega(q)) / (E(q))
* **уширения/времена жизни**: (\Gamma(q,T)), (\tau(q,T)), (Q^{-1}) (если есть)
* **энергосрезы/спектры**: интенсивность (I(\Delta E)) при фиксированном (Q,T) (для фитинга (\Gamma))
* **интегральные интенсивности/параметры порядка**: (I(T)), ((I/T)^{-1}(T)), и т.п.
* **теплоёмкость/параметры Дебая**: (C(T)), (\Theta_D), (если статья про это)

**Не извлекать**: выводы, модели, теорию, параметры подгонки *если они не являются прямой публикацией результатов измерений* (исключение: когда авторы прямо дают численные “measured …” в тексте/таблицах).

---

## 1) Унифицированный алгоритм работы (для каждой статьи)

### 1.1 Инвентаризация (обязательно перед оцифровкой)

1. Пройти PDF и выписать **все**:

* номера **Figures** и их подписи (caption),
* номера **Tables** и их заголовки,
* где в тексте написано “measured”, “observed”, “experimental”, “data”.

2. Для каждого Figure/Table назначить статус:

* `EXPERIMENT` — это измеренные точки/значения;
* `DERIVED` — вычислено из модели/фита, но всё ещё может быть сравниваемо (например, (\Gamma) получено фитингом спектра и приведено числом);
* `SCHEMATIC/NOT_DATA` — схемы, иллюстрации, геометрия, векторные картинки без чисел.

**Выход инвентаризации**: `paper_map.yaml` (список объектов с кратким смыслом и статусом).

---

### 1.2 Оцифровка графиков (CV-якоря → шкалы → серии)

Для каждого `EXPERIMENT/DERIVED` графика:

**Шаг A — CV-якоря (обязательно сохранить):**

* bbox области графика,
* оси (линии), сетка (если есть),
* подписи осей (что по x/y и единицы),
* ticks (минимум 2 по x и 2 по y),
* легенда (серии → маркер/цвет → физический смысл),
* подпись рисунка (caption) как текст.

**Шаг B — калибровка шкал:**

* получить преобразование `pixel → data` по tick-меткам,
* если шкала логарифмическая — явно это отметить (`scale_x/log`, `scale_y/log`).

**Шаг C — извлечение серий:**

* выделить маркеры/кривые по цвету/форме,
* каждой точке присвоить:

  * `series_id` (например `T=T0+55C`, `Ein=38meV`, `branch=TA`, …),
  * `x`, `y`,
  * `x_err`, `y_err` (если есть error bars),
  * `quality_flag` (OK / uncertain / bad).

**Шаг D — самопроверка графика:**

* “reproject check”: наложить извлечённые точки обратно на изображение и посчитать среднюю пиксельную ошибку.
* если ошибка выше порога — пометить `needs_manual`.

---

### 1.3 Извлечение таблиц

Для каждой таблицы `EXPERIMENT/DERIVED`:

* распознать структуру (строки/колонки),
* сохранить CSV ровно в опубликованных единицах,
* отдельным файлом сохранить `table_notes.md` (что означает каждая колонка).

**Проверка таблицы:**

* сверить количество строк с оригиналом,
* сверить 3–5 случайных ячеек “глазами” (модель должна вывести список проверяемых позиций).

---

### 1.4 Формат выходных данных (строго)

Для каждой статьи:

```
crystal_experiment_data/
  <paper_key>/
    paper_map.yaml
    figures/
      figXX_<shortname>.csv
      figXX_<shortname>.meta.yaml
      figXX_<shortname>_anchors.png   (с отмеченными осями/легендой)
      figXX_<shortname>_reproj.png    (точки поверх графика)
    tables/
      tableY_<shortname>.csv
      tableY_<shortname>.meta.yaml
    README.md
    QA_REPORT.md
```

## 2) Что конкретно извлекать “по смыслу” из каждой статьи

Ниже — **минимальный список объектов**, которые *имеют смысл* как сравниваемые экспериментальные данные.

---

## A) Cowley (1963) — нейтронное рассеяние, фононы, времена жизни / уширения (NaI и др.)

**Извлекать обязательно:**

1. **Таблицы с численными фононными параметрами** (частоты/энергии, ширины линий, времена жизни, температуры, точки BZ/ветви).

   * Любая таблица вида: `q, branch, ω (или E), Γ (или τ), T`.
2. **Графики уширения/затухания** (типично (\Gamma) vs (q) или (\Gamma) vs (\omega), либо (\tau) vs (T)).
3. **Дисперсионные кривые** (\omega(q)), если в статье они даны точками (не только линиями “guide to the eye”).

**Если есть профили спектров (I(\Delta E))**:

* оцифровать сами профили для нескольких температур/точек (это исходник для фитинга (\Gamma)).

**Проверка полноты для Cowley:**

* должно получиться: минимум один CSV из таблицы + минимум один CSV из фигуры с уширениями/или дисперсией.
* для каждого материала (например NaI) — отдельная серия/маркер в данных.

---

## B) Dolling (1962) — фононные дисперсии/температурные зависимости (кристалл, нейтроны)

**Извлекать обязательно:**

1. **Фононные дисперсии** (\omega(q)) / (E(q)) по направлениям (обычно (\Gamma\to X), (\Gamma\to L), (\Gamma\to K) и т.п.).

   * Каждая ветвь (LA/TA/LO/TO) — отдельная `series_id`.
   * Если на одном графике несколько температур — серия должна включать `T=...`.
2. Если есть: **ширины линий / времена жизни** vs (q,T) — извлекать так же, как в Cowley.
3. Если авторы дают **численные значения в таблицах** (частоты в высокосимметричных точках, elastic constants из эксперимента) — извлекать как таблицы.

**Проверка полноты Dolling:**

* для каждого графика дисперсии: извлечены все ветви, которые явно отмечены точками/маркерами.
* диапазон (q) соответствует подписи (например 0→1 в reduced units).

---

## C) Axe (1970) — кварц (α/β), мягкая мода, спектры, интегральные интенсивности

По тем страницам/рисункам, которые уже у тебя есть в превью, **экспериментальные данные здесь — это конкретные фигуры и 1 таблица**.

### Axe: извлекать обязательно (EXPERIMENT/DERIVED)

**Fig.2**
Смысл: энергетические спектры (I(\Delta E)) при фиксированном (Q) для β-quartz при нескольких температурах.
Что извлечь:

* `x = ΔE (meV)`
* `y = neutron counts / intensity`
* серия = температура (например `T0+4`, `T0+55`, `T0+160`)
* если есть отдельная кривая фона — тоже отдельной серией.

**Fig.3**
Смысл: дисперсия низкоэнергетического возбуждения/моды в β-quartz вдоль направления (q=(\xi,0,0)) (ветви).
Что извлечь:

* `x = ξ` (или reduced q)
* `y = E (meV)`
* серия = (ветвь + температура).
  Важно: если на графике два температурных набора точек — разделить.

**Fig.4**
Смысл: энергетические спектры (I(\Delta E)) около заданного отражения (пример: (1,0,3)) при нескольких температурах (β-quartz).
Что извлечь:

* `ΔE (meV)` vs `intensity`
* серия = температура (`T0+18`, `T0+55`, `T0+214`, …)
  Это ключевые “сырые” данные для последующего извлечения (\Gamma(T)).

**Fig.5**
Смысл: температурная зависимость интегральной интенсивности (в графике вида ((I/T)^{-1}) vs (T)), причём серии различаются по энергии падающих нейтронов.
Что извлечь:

* `x = T (K)`
* `y = (I/T)^(-1)` (как подписано на оси)
* серия = `Ein=38 meV` и `Ein=19 meV` (или как в легенде).

**Table I (на странице с таблицей)**
Смысл: численные результаты по моде/смещениям, где часть значений обозначена как экспериментальные/наблюдаемые (“present study” и т.п.).
Что извлечь:

* только те колонки, которые являются **результатом измерений** (если таблица смешивает “measured” и “calculated” — разделить статусом).
* обязательно сохранить единицы (см/^-1, Å, безразмерные).

**Fig.8, Fig.9 (если используешь α-quartz часть)**
Смысл:

* Fig.8 — сравнение наблюдаемых интенсивностей для 208 cm^-1 моды (дискретные точки по наборам (Q)).
* Fig.9 — спектры (I(\Delta E)) при двух температурах (20°C и 400°C) около (006).
  Что извлечь:
* Fig.8: таблицу вида `Q_point_label, I_obs (±err)` (лучше вручную как “категориальную ось”).
* Fig.9: `ΔE` vs `intensity` для двух температур.

### Axe: критерий полноты

Минимальный обязательный комплект CSV:

* fig2 spectra
* fig3 dispersion
* fig4 spectra
* fig5 (I/T)^-1 vs T
* table I (экспериментальные колонки)

Если что-то не распознаётся (плохие ticks/легенда) — обязателен `QA_REPORT.md` с причинами и приложенными anchors-картинками.

## D) McKnight et al. (2008) — теплоёмкость / Debye-параметры (по смыслу)

(Тут модель должна подтвердить по PDF, какие именно измеряемые величины представлены.)

**Что искать и извлекать:**

1. Таблицы/графики **теплоёмкости**: (C_p(T)) или (C_v(T)).
2. Любые таблицы с **(\Theta_D)** (Debye temperature) и как она получена.
3. Если есть **параметры решётки/плотность/скорости звука** как экспериментальные — извлекать таблицами (они потом идут в классические расчёты и в перевод к Θ-параметрам).
4. Если есть сравнение разных материалов — каждая кривая/серия должна быть помечена материалом.

**Критерий полноты McKnight:**

* минимум один полный набор (C(T)) (таблица или кривая) + таблица/кривая (\Theta_D) (если присутствует в статье).
